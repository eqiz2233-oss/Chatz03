Chatz — Railway production deploy
====================================

ตัวอย่าง URL: https://chatz02-production.up.railway.app
URL นี้คือทั้ง UI และ webhook endpoint (Express เสิร์ฟ /api + dist/ พอร์ตเดียวกัน)

ไฟล์ config ที่รากโปรเจกต์ (มีอยู่แล้ว)
---------------------------------------
  Dockerfile       → multi-stage build (npm ci → npm run build → runtime image)
  .dockerignore    → ลด context ไม่ส่ง node_modules / .env
  railway.toml     → builder = DOCKERFILE + start = npm start + healthcheck /api/health

Variables (ตั้งใน Railway → service → Variables)
-------------------------------------------------
  PUBLIC_BASE_URL=https://chatz02-production.up.railway.app

  # LINE
  LINE_CHANNEL_SECRET=...
  LINE_CHANNEL_ACCESS_TOKEN=...

  # Meta (Facebook + Instagram)
  FB_APP_ID=...
  FB_APP_SECRET=...
  FB_VERIFY_TOKEN=chatz-verify-... (อะไรก็ได้ — ใส่ค่าเดียวกันใน Meta App)
  # FB_GRAPH_VERSION=v21.0   (optional)

  # ไม่ต้องใส่ FB_PAGE_ACCESS_TOKEN — ระบบเก็บผ่าน OAuth ใน server/integrations.json อัตโนมัติ

URL ที่ต้องตั้งใน Meta App Dashboard (ทำครั้งเดียว)
--------------------------------------------------
  Webhook — ต้องทำแยก 2 ที่ (ทั้งคู่ใช้ URL เดียวกันได้)
    Messenger (Facebook Page):
      Callback URL : https://<โดเมน-railway>/api/fb/webhook
      Verify Token : ค่าเดียวกับ FB_VERIFY_TOKEN
      ในตาราง Webhook fields ให้ Subscribe อย่างน้อย: messages
    Instagram (สำคัญ — ถ้าไม่ทำ IG จะไม่เข้า):
      ในแอป Meta → สินค้า "Instagram" → หัวข้อ Webhooks
      ใส่ Callback URL เดียวกัน: https://<โดเมน-railway>/api/fb/webhook
      Verify + Subscribe field **messages** (และอื่นๆ ตามต้องการ)
      Meta ระบุว่า IG webhook ต้องตั้งใน Dashboard นี้ — ไม่ได้มาจาก POST Page subscribed_apps อย่างเดียว

  ถ้า Meta ขึ้น "callback URL or verify token couldn't be validated"
    → เกือบทั้งหมดคือ FB_VERIFY_TOKEN ใน Railway ไม่ตรงกับช่อง Verify Token ใน Meta
    → ตรวจว่ามีตัวแปร FB_VERIFY_TOKEN ใน Railway จริง (ไม่ว่าง) ไม่มี space หัวท้าย
    → ทดสอบด้วยเบราว์เซอร์ (แทน YOUR_TOKEN ด้วยค่าเดียวกับ Railway):
      https://chatz02-production.up.railway.app/api/fb/webhook?hub.mode=subscribe&hub.verify_token=YOUR_TOKEN&hub.challenge=ok
      ถ้าถูกต้องหน้าจะแสดงแค่คำว่า ok

  Facebook Login
    Valid OAuth Redirect URIs:
      https://chatz02-production.up.railway.app/api/fb/oauth/callback

  App Settings → Basic
    Privacy Policy URL          : https://chatz02-production.up.railway.app/api/privacy
    Data Deletion Callback URL  : https://chatz02-production.up.railway.app/api/fb/data-deletion

Webhook ของ LINE (ตั้งใน LINE Developers Console)
--------------------------------------------------
  Webhook URL : https://chatz02-production.up.railway.app/api/line/webhook
  เปิด Use webhook + กด Verify

ที่ผู้ใช้ปลายทางต้องทำ
---------------------
  1. ตั้ง Webhook ทั้ง Messenger และ Instagram ใน Meta (ดูหัวข้อ Webhook ด้านบน)
  2. เปิด UI → Settings → Integrations → Connect Facebook & Instagram → Login → เลือกเพจ
     (ขั้นนี้ subscribe ฝั่ง Facebook Page — คู่กับ Messenger)
  3. IG: เพจต้องผูก Instagram Professional/Business; แอปต้องมี instagram_manage_messages (App Review ถ้าผู้ส่งไม่ใช่ role บนแอป)
  4. Meta กำหนดว่าแอปต้องอยู่สถานะ Published จึงจะรับ webhook บางประเภทจาก Instagram

API endpoints หลัก
------------------
  GET  /api/health
  POST /api/messages/send       → unified (รับ line: / fb: / ig: prefix)
  POST /api/fb/webhook          → รองรับทั้ง object=page และ object=instagram
  POST /api/line/webhook
  GET  /api/fb/oauth/start      → เริ่ม Facebook OAuth
  GET  /api/fb/oauth/callback   → callback (วาง URL นี้ใน Meta)
  GET  /api/privacy             → privacy policy (วาง URL นี้ใน Meta)
  POST /api/fb/data-deletion    → data-deletion callback (วางใน Meta)

ห้าม commit .env ขึ้น git — ตั้งใน Railway Variables เท่านั้น
ถ้าเปลี่ยน env ต้อง redeploy ใหม่ (Railway auto-redeploy เมื่อ Variables เปลี่ยน)
